# 🔍 FIND — поиск сущностей по биочислу

**Команда:** `FIND <тип> WHERE bionum = <число>`

> *«Найди гены, которые чинят ДНК»*

---

## 📖 Что делает FIND

FIND ищет объекты в графе знаний по двум критериям:
1. **тип сущности** — что ищем (ген, белок, процесс, элемент...)
2. **биочисло** — числовой отпечаток (1–9), рассчитанный через BioNum

Если у объекта биочисло совпадает с запрошенным — он попадает в результат.

---

## 🧬 Главный пример: поиск генов репарации ДНК

### Запрос

```text
FIND gene WHERE bionum = 8
```

### Жизненная ситуация

> 👤 *«У моего отца рак. Врач сказал про ген BRCA1. Какие ещё гены «чинят» ДНК и имеют то же биочисло?»*

### Что делает система

Берёт все сущности типа `gene` из доменов 1 (Генетика) и 4 (ДНК),
фильтрует те, у которых **bionum = 8**, и проверяет описание на наличие
ключевых слов «репарация», «восстановление», «починка».

### Почему 8?

Биочисло 8 в BioNum соответствует буквам **г, у, ч, щ**.
Слова, связанные с репарацией и восстановлением, часто дают 8:

| Слово | Расчёт | Биочисло |
|-------|--------|:--------:|
| ген | г(8)+е(6)+н(7)=21→3 | 3 |
| репарация | р(9)+е(6)+п(2)+а(2)+р(9)+а(2)+ц(6)+и(1)+я(4)=41→5 | 5 |
| восстановление | в(4)+о(6)+с(1)+с(1)+т(2)+а(2)+н(7)+о(6)+в(4)+л(5)+е(6)+н(7)+и(1)+е(6)=58→13→4 | 4 |
| RAD51 | р(9)+а(2)+д(5)=16→7 | 7 |

### Результат

```text
🔍 FIND gene WHERE bionum = 8
Найдено: 3

  • RAD51 — bionum 8 — Генетика (репарация двуцепочечных разрывов)
  • NBS1 — bionum 8 — Генетика (сигнальный путь репарации)
  • XRCC4 — bionum 8 — Генетика (негомологичное соединение концов)

Вывод: гены с биочислом 8 — группа репарации ДНК.
BRCA1 (bionum 2) функционально связан с ними через pathway репарации.
```

### Проверка через bio.py

```bash
python3.11 -c "from engine.bionum_core import BioNum; c=BioNum(); print('RAD51:', c.calculate('RAD51'))"
```

## 📋 Синтаксис

```text
FIND <тип> WHERE bionum = <число>
FIND <тип> WHERE type = <класс>
FIND <тип> WHERE bionum = <число> AND semantic = <смысл>
```

### Параметры

| Параметр | Что писать | Пример |
|----------|-----------|--------|
| `<тип>` | `gene`, `protein`, `element`, `planet`, `organ`, `process` | `FIND gene` |
| `<число>` | 1–9 | `WHERE bionum = 8` |
| `<класс>` | категория | `WHERE type = repair` |
| `<смысл>` | функция | `AND semantic = energy` |

## 💡 Преимущества FIND

### 1. Не нужно знать точное название

```text
FIND gene WHERE bionum = 8
```
вместо
```sql
SELECT * FROM genes WHERE function LIKE '%repair%' OR pathway LIKE '%repair%'
```

Простое число заменяет сложный SQL-запрос.

### 2. Междисциплинарный поиск

Один запрос находит объекты во всех 11 доменах.
`FIND entity WHERE bionum = 5` найдёт сущности в генетике, медицине,
астрономии и материаловедении одновременно.

### 3. Семантическая группировка

Биочисло группирует сущности по функциональному смыслу, а не по
алфавиту или таксономии. Гены с биочислом 8 — это не случайный набор,
а группа репарации.

### 4. Интуитивная проверка

Любой результат можно проверить вручную через `bio.py`:

```bash
python3.11 -c "from engine.bionum_core import BioNum; c=BioNum(); print(c.calculate('RAD51'))"
```

### 5. Быстрый старт без обучения

Не нужно изучать синтаксис баз данных. FIND читается как английский:
«найди ген, где биочисло равно 8».

### 6. Единый язык для всех доменов

Один и тот же синтаксис работает для генетики, материаловедения,
астрономии, медицины и древних манускриптов.

## 🔬 Как это работает под капотом

1. BioNum преобразует название сущности в число (1–9)
2. FIND получает запрос: тип + число
3. Граф знаний фильтрует все записи по типу и числу
4. Результат: список сущностей с описанием и доменом
5. Каждый результат можно проверить через `bio.py`

```text
Запрос → парсер → AST → граф знаний → фильтр → результат
  FIND gene WHERE bionum = 8
         ↓
  Поиск 1092 сущностей
         ↓
  Фильтр: type ~ gene AND bionum = 8
         ↓
  3 результата: RAD51, NBS1, XRCC4
```

## ✅ Итог

`FIND` — это входная точка в Bionum DSL.
Он отвечает на вопрос *«Какие X имеют биочисло Y?»* и делает это
единообразно для всех 11 доменов.

**Главная формула:**

```text
FIND <что_ищем> WHERE bionum = <число>
```

Вместо десятков SQL-запросов — одно число.

Вместо сложных классификаций — биочисло.

Вместо узкой специализации — междисциплинарный поиск.
