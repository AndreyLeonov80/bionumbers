## 📋 Что такое Bionum DSL

Bionum DSL — это предметно-ориентированный язык (DSL) для поиска и анализа
связей между сущностями из 11 научных доменов. Каждая сущность имеет
**биочисло** (1–9), рассчитанное через лингво-числовое преобразование
русского слова.

```text
Каждая буква → число → сумма → редукция до 1 цифры
    г(8)+е(6)+н(7) = 21 → 2+1 = 3
```

**Суть:** у объектов с одинаковым биочислом есть функциональный резонанс —
скрытая связь между разными областями знаний.

## 🎮 10 команд DSL

### 1. FIND — поиск сущностей

**Синтаксис:**
```text
FIND <тип> WHERE bionum = <число>
FIND <тип> WHERE type = <класс>
FIND <тип> WHERE bionum = <число> AND semantic = <смысл>
```

**Пример:**
```text
FIND gene WHERE bionum = 8
```
```text
Результат: найдены гены с биочислом 8 — RAD51, NBS1, XRCC4
Все они связаны с репарацией ДНК.
```

**Жизненный сценарий:**
> 👤 *«У моего отца рак. Какие гены «чинят» ДНК и имеют биочисло 8?»*

---

### 2. LINKS OF — связи сущности

**Синтаксис:**
```text
LINKS OF <сущность>
INCOMING TO <сущность>
```

**Пример:**
```text
LINKS OF теломера
```
```text
Результат: теломера связана с TERT, TERC, Shelterin, TRF1, TRF2, POT1
Биочисло теломеры — 7. Резонанс: старение, генетика, медицина.
```

**Жизненный сценарий:**
> 👤 *«С чем связана теломера? Какие гены влияют на длину теломер?»*

---

### 3. PATH — причинно-следственный путь

**Синтаксис:**
```text
PATH <от> -> <до> [MAX <шаги>]
PATH <от> -> <до> USING <отношение1>, <отношение2>
```

**Пример:**
```text
PATH хромосома -> мутация
```
```text
Результат: хромосома(7) → репликация(?) → повреждение_ДНК(?) → мутация(?)
Цепочка из 4 шагов через процессы репликации и репарации.
```

**Жизненный сценарий:**
> 👤 *«Как повреждение ДНК приводит к раку? Построй цепочку от хромосомы до мутации.»*

---

### 4. CLUSTER — кластеризация по биочислу

**Синтаксис:**
```text
CLUSTER WHERE bionum = <число>
CLUSTER AROUND <сущность>
PATTERN FIND [<число> -> <число> -> <число>]
```

**Пример:**
```text
CLUSTER WHERE bionum = 7
```
```text
Результат: 99 сущностей с биочислом 7 из всех 11 доменов.
Включает: фенилкетонурия, ДНК-полимераза, CR-диета, ATM-repair, ...
```

**Жизненный сценарий:**
> 👤 *«Какие процессы имеют биочисло 7? Может, они связаны между собой?»*

---

### 5. TRANSITIONS — переходы биочисла

**Синтаксис:**
```text
TRANSITIONS FROM <число>
TRANSITION MATRIX
MOST COMMON TRANSITION
```

**Пример:**
```text
TRANSITIONS FROM 7
```
```text
Результат:
  7 → 4 (42% связей)
  7 → 8 (31% связей)
  7 → 9 (18% связей)
  Вывод: биочисло 7 чаще всего переходит в 4.
```

**Жизненный сценарий:**
> 👤 *«Что происходит после активации гена с биочислом 7? Куда система переходит дальше?»*

---

### 6. ANOMALIES — аномалии и выбросы

**Синтаксис:**
```text
ANOMALIES IN <домен>
RARE bionum IN <домен>
OUTLIERS WHERE bionum = <число>
```

**Пример:**
```text
ANOMALIES IN генетика
```
```text
Результат: в домене «Генетика» найдены аномалии —
сущности с биочислами, сильно отклоняющимися от среднего (4.2).
  • Синдром Дауна — bionum 2 (отклонение 2.2)
  • мтДНК — bionum 8 (отклонение 3.8)
```

**Жизненный сценарий:**
> 👤 *«Есть ли аномальные гены в генетике? Какие сущности выбиваются из общего ряда?»*

---

### 7. RESONANCE — междоменный резонанс

**Синтаксис:**
```text
RESONANCE OF <сущность>
RESONANCE BETWEEN <сущность1> AND <сущность2>
MAX RESONANCE IN <домен>
```

**Пример:**
```text
RESONANCE OF теломера
```
```text
Результат: теломера (bionum=7) резонирует:
  📍 Старение: длина_теломер, TERT, TERC
  📍 Генетика: теломераза, Shelterin
  📍 Медицина: возрастные болезни
  📍 ДНК: хроматин, центромера
```

**Жизненный сценарий:**
> 👤 *«Теломера связана со старением. А с чем ещё? Есть ли резонанс с другими областями?»*

### 8. HYPOTHESIZE — генерация гипотез

**Синтаксис:**
```text
HYPOTHESIZE ON <тема>
HYPOTHESIZE ON <тема> WHERE confidence > 0.8
HYPOTHESIZE FROM [<число> -> <число> -> <число>]
```

**Пример:**
```text
HYPOTHESIZE ON старение
```
```text
Результат:
  🔮 теломераза — достоверность 0.85
    Причина: общее биочисло 7 со старением
  🔮 NAD — достоверность 0.72
    Причина: переход 7 → 4 (42%)
  🔮 sirtuin — достоверность 0.68
    Причина: кластер биочисла 7
```

**Жизненный сценарий:**
> 👤 *«Я изучаю старение. Какие гипотезы можно выдвинуть на основе биочисел?»*

### 9. EXPLAIN — полное объяснение

**Синтаксис:**
```text
EXPLAIN <сущность>
WHY <сущность1> -> <сущность2>
```

**Пример:**
```text
EXPLAIN хромосома
```
```text
📋 хромосома (chromosome)
Биочисло: 5
Домен: ДНК/хромосомы/гены → Структура хромосом
Описание: Структура хромосомы, носитель генетической информации

Связи:
  🔗 центромера (bionum ?)
  🔗 теломера (bionum 7)
  🔗 хроматида (bionum ?)

Резонанс: 7 сущностей в 4 доменах
Кластер: 100 сущностей с биочислом 5
```

**Жизненный сценарий:**
> 👤 *«Расскажи всё о хромосоме простыми словами: что это, с чем связано, где ещё встречается такое же биочисло.»*

### 10. DISCOVER — открытие закономерностей

**Синтаксис:**
```text
DISCOVER <паттерн>
```

**Паттерны:**
- `CONSTANTS` — распределение биочисел по системе
- `CYCLES` — циклические паттерны доменов
- `CLUSTERS` — размеры кластеров

**Пример:**
```text
DISCOVER CONSTANTS
```
```text
📊 Распределение биочисел:
  bionum 1: 106 сущностей (9.7%)
  bionum 2: 110 сущностей (10.1%)
  bionum 3: 120 сущностей (11.0%)
  bionum 4: 98 сущностей (9.0%)
  bionum 5: 100 сущностей (9.2%)
  bionum 6: 111 сущностей (10.2%)
  bionum 7: 99 сущностей (9.1%)
  bionum 8: 96 сущностей (8.8%)
  bionum 9: 83 сущностей (7.6%)

Вывод: распределение равномерное, небольшие пики на 3 и 6.
```

**Жизненный сценарий:**
> 👤 *«Есть ли скрытые закономерности в распределении биочисел по доменам?»*

## 💡 Преимущества Bionum DSL

### 1. Простота

Не нужно знать SQL, Python или биоинформатику. Запросы выглядят как
естественный язык:

```text
EXPLAIN хромосома         вместо  SELECT * FROM entities WHERE name = 'chromosome'
RESONANCE OF теломера     вместо  JOIN ... WHERE bionum = 7
```

### 2. Междисциплинарность

Одно биочисло объединяет сущности из разных доменов. Это позволяет
находить неочевидные связи:

> **Графен (биочисло 9)** резонирует с **нейросетью (9)** и
> **марфаном (9)** — скрытая связь между материаловедением,
> ИИ и генетикой.

### 3. Предсказательность

Гипотезы и переходы позволяют прогнозировать развитие систем:

> Если биочисло 7 чаще переходит в 4 (42%), то активация гена с
> биочислом 7 с высокой вероятностью запустит процесс с биочислом 4.

### 4. Обнаружение аномалий

Система находит сущности, чьи биочисла выбиваются из среднего
по домену — это помогает выявлять редкие и необычные объекты.

### 5. Наглядность

Каждый результат сопровождается:
- биочислом (цифра 1–9)
- расчётом (буквы → числа → сумма → редукция)
- визуальной группировкой по доменам

### 6. Полнота

1092 сущности в 11 доменах, 10920 use-case кейсов —
вся система верифицирована через `bio.py` с нулевой ошибкой.

### 7. Расширяемость

Добавление новой сущности:
```python

("криоэлектроника", 2, "2.1", "cryoelectronics", "...", "...")

# 2. Пересобрать
python3.11 -c "from engine.build_spec import build_csv; build_csv()"

# 3. Перегенерировать use-case
python3.11 engine/batch_generator.py

# 4. Готово — сущность доступна во всех 10 командах DSL
```

## 🔮 Философия

Bionum DSL — это лингво-числовой мост между научными доменами.

**Идея:** если два объекта имеют одинаковое биочисло, между ними есть
функциональная связь, даже если они находятся в разных областях знаний.

**Применение:**
- поиск неочевидных связей между открытиями
- генерация гипотез для междисциплинарных исследований
- образовательный инструмент: понимание сложного через простое биочисло
- диагностика: аномалии и выбросы указывают на редкие явления
- прогнозирование: переходы биочисел предсказывают развитие систем
